React Server Components: Qué Son y Cuándo Usarlos

React sigue evolucionando y una de las incorporaciones más disruptivas en los últimos años son los React Server Components (RSC). En este artículo te explicamos qué son, cómo funcionan y cuándo conviene usarlos.

¿Qué son los React Server Components?

Los React Server Components permiten renderizar componentes directamente en el servidor, sin incluir su lógica en el JavaScript que se envía al cliente. Esto significa que el componente se procesa en el backend y el resultado HTML se entrega al navegador.

"Con RSC, el cliente recibe solo lo que necesita ver. Nada más, nada menos."

Diferencias clave con los componentes tradicionales

  • No tienen efectos secundarios (no usan useEffect).
  • No manejan estado en el cliente (useState no es válido).
  • Permiten acceder a recursos del servidor directamente (como base de datos o APIs internas).
  • No aumentan el tamaño del bundle en el navegador.

Ventajas de usar RSC

Al renderizar componentes en el servidor, obtenés mejores tiempos de carga, menos JavaScript en el cliente, y un rendimiento superior, especialmente en apps grandes o con mucha lógica de datos.

Casos de uso ideales

  • Listados de productos o contenido estático que cambia con poca frecuencia.
  • Tablas, dashboards y gráficas con muchos datos.
  • Autenticación y renderizado condicional según permisos de usuario.
  • Renderizado de contenido basado en consultas a base de datos.

¿Cómo se usan?

Los RSC están disponibles en Next.js usando el App Router (no el tradicional Pages Router). Simplemente creás un componente con la extensión .server.js o marcás la función como async y la exportás desde un archivo del servidor. 


// ejemplo.server.js
export default async function ProductosDestacados() {
  const productos = await obtenerProductosDesdeDB();
  return (
    <ul>
      {productos.map(p => <li key={p.id}>{p.nombre}</li>)}
    </ul>
  );
}

¿Cuándo evitar RSC?

Si tu componente depende de interacciones del usuario en tiempo real, como formularios dinámicos o animaciones, lo mejor es usar componentes cliente. También evitá RSC si necesitás hooks como useEffect o useState.

Conclusión

React Server Components marcan una nueva era para React y el desarrollo web moderno. Son ideales para apps donde el rendimiento y la eficiencia del bundle son críticos.

Aprender a combinar RSC con componentes cliente será clave para desarrollar aplicaciones escalables y rápidas en 2025. ¿Ya empezaste a usarlos?
